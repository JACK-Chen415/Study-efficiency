# AGENTS.md

本文件适用于本仓库全部目录。后续 Codex 执行任务时必须遵守以下规则。

## 工作前检查

1. 开始修改前先阅读 `docs/PLAN.md` 与 `docs/TASKS.md`；涉及接口或数据时同步阅读 `docs/API_SPEC.md` 与 `docs/DATA_DICTIONARY.md`。
2. 检查当前工作区已有文件和改动，保留用户内容，不覆盖或回退无关修改。
3. 明确当前 goal 的修改范围、完成标准、验证方式和禁止事项。

## Milestone 规则

1. 一次只完成一个明确 milestone，不将多个阶段捆绑实施。
2. 以 P0 主链路为优先，不提前实现 `PLAN.md` 中延期功能。
3. 不进行与当前 milestone 无关的大规模重构、依赖升级或目录改造。
4. 当范围、契约或优先级变动时，先同步相应文档与任务状态。

## 安全与数据规则

1. 不能提交真实密钥、密码、token、连接串或含敏感信息的 `.env` 文件；仅允许提供脱敏示例配置。
2. 不能乱删文件、用户数据、数据库内容或现有实验记录。确需清理时，先暂停并取得用户确认。
3. 不能修改系统全局环境或安装大批依赖，除非用户明确批准。
4. 真实采集数据与模拟测试数据必须区分记录；任何报告不得将模拟数据描述为真实结论。
5. 手机传感器信息属于辅助输入，权限拒绝或不可用不得阻断学习记录主流程。

## 修改后验证

1. 每次修改后必须运行与改动相对应的验证，并在交付说明中报告结果。
2. 仅修改文档时，验证文件存在、Markdown 关键章节与任务勾选状态正确。
3. 后端代码修改后，运行 `pytest`；如尚无测试，至少启动 FastAPI 并检查 `/docs` 可访问，同时说明测试缺口。
4. 前端代码修改后，运行 `npm run build`。
5. 数据库模型或 API 变更后，同步更新 `docs/DATA_DICTIONARY.md` 与 `docs/API_SPEC.md`。
6. 模型训练后，将样本、特征、指标和限制写入 `docs/EXPERIMENT_LOG.md`。

## 阻塞处理

遇到缺少必要输入、无法安全验证、权限/依赖问题、契约冲突或可能破坏用户数据的操作时，立即暂停该 milestone，清楚报告阻塞原因、已完成内容和需要用户决定的下一步，不擅自扩大范围绕过问题。
