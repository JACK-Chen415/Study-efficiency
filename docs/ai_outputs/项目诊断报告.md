# 项目诊断报告

## 1. 诊断边界

本报告基于仓库内 `README.md`、`docs/PLAN.md`、`docs/TASKS.md`、`docs/API_SPEC.md`、`docs/DATA_DICTIONARY.md`、`docs/DATA_COLLECTION_PROTOCOL.md` 与 `docs/EXPERIMENT_LOG.md` 整理，面向课程项目、科研训练和答辩准备场景。

重要边界：

- 当前项目可以作为“大学生学习效率预测系统”的 MVP 原型进行展示。
- 当前已有 demo/mock 数据链路和训练前数据准备说明，但不能把 demo/mock 数据写成真实实验结果。
- 截至现有实验日志，尚未训练正式模型、尚未生成真实模型指标，也不能声称系统已经得到真实预测结论。
- 手机运动数据是辅助输入，不等同于对专注度或学习效果的直接测量。

## 2. 项目定位

项目面向大学生个人自习场景，用手机浏览器作为轻量入口，记录学习起止时间、自报告状态和可选手机运动聚合特征，再为后续低/中/高学习效率三分类建模和可视化看板提供数据基础。

对大二物联网专业学生而言，本项目的训练价值主要体现在：

| 维度 | 体现方式 |
| --- | --- |
| 物联网感知 | 使用手机浏览器 `DeviceMotionEvent` 获取轻量运动聚合特征，不额外依赖专用硬件。 |
| 全栈系统 | Vue 3 手机端、FastAPI 后端、数据库、训练数据导出脚本共同构成闭环。 |
| 数据建模 | 将自报告量表、时间、地点、任务类型和运动特征整理为机器学习表格数据。 |
| 科研意识 | 区分真实数据、测试数据和模拟数据，说明样本量、隐私和数据质量限制。 |
| 答辩表达 | 可以讲清楚“为什么采集、采集什么、如何建模、风险在哪里”。 |

## 4. 数据资产诊断

项目数据应分为三类管理：

| 数据类型 | 作用 | 是否可写入真实结论 |
| --- | --- | --- |
| 真实数据 | 参与者完成真实学习后提交的记录 | 可以进入正式统计和模型训练，但必须经过质量检查。 |
| 测试数据 | 开发调试接口、页面和流程时产生的记录 | 不能作为真实样本，不能计入正式样本量。 |
| demo/mock 数据 | 人工构造或 seed 脚本生成的数据，用于演示流程 | 只能证明系统链路可运行，不能证明模型有效。 |

建议在采集、导出和报告中持续保留以下说明：

- `abandoned` 记录不进入训练，也不能解释为低效率样本。
- 删除记录归档用于追溯，不作为默认训练数据来源。
- 缺失运动特征不直接排除；训练导出可用 0 填充，并用 `motion_available` 标记缺失机制。
- `efficiency_score` 是主观自评标签，不是考试成绩、客观绩点或真实学习成果。
- 低样本阶段只能做流程验证或限制性探索，不能形成普适结论。

## 5. 主要风险

| 风险 | 表现 | 建议应对 |
| --- | --- | --- |
| 样本量不足 | 少于 30 条真实有效 completed 记录时模型训练意义有限 | 先做采集和质量检查；展示时说明“尚处原型与数据积累阶段”。 |
| 标签不均衡 | 低/中/高某一类样本很少，指标容易失真 | 统计标签分布，必要时延长采集，不只看 Accuracy。 |
| 自报告偏差 | 参与者可能凭感觉或迎合系统填写 | 在采集说明中强调按真实体验填写，并标记明显乱填记录。 |
| 运动传感器缺失 | 浏览器不支持、权限拒绝、后台暂停或上传失败 | 保持主流程可用，把运动数据定位为可选辅助特征。 |
| 隐私表达过度 | 自选昵称不等于绝对匿名 | 使用“去标识化采集”，不收集姓名、学号、手机号、精确定位等信息。 |
| 结论夸大 | 用 demo/mock 或小样本指标证明模型有效 | 报告中明确“当前无真实预测结论”，真实训练后再记录指标。 |

## 6. 下一步建议

1. 继续按 `DATA_COLLECTION_PROTOCOL.md` 采集真实学习记录，先完成本周 20 条，再向 50 条有效样本推进。
2. 每次导出训练数据前统计真实、测试、mock/demo 的数量，避免混入。
3. 训练前填写 `EXPERIMENT_LOG.md` 的数据就绪检查模板，重点写明样本数、标签分布、运动缺失率和排除规则。
4. 满足最低条件后再训练 RandomForest，并与多数类 baseline 对比；不只汇报 Accuracy，还要汇报 Precision、Recall、F1 和混淆矩阵。
5. 答辩中将项目表述为“基于手机轻量感知和自报告数据的课程级原型系统”，避免说成成熟预测产品。

## 7. 可直接使用的答辩定位

本项目不是试图用手机传感器直接判断一个人是否认真学习，而是构建一个轻量化学习记录系统：通过手机端打卡、自报告量表和可选运动聚合特征，形成可清洗、可训练、可解释的数据集。当前阶段重点完成采集闭环和训练前数据准备；正式模型结论需要在真实有效样本达到条件后再给出。
